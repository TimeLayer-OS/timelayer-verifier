# TimeLayer — оффлайн-верификатор

[English](README.md) · **Русский**

Маленький самодостаточный инструмент, который **проверяет квитанцию TimeLayer офлайн** — без
сети, без обращения к реестру и без доверия к нам. Квитанция — это **пара файлов**, несущая
всё необходимое для проверки; верификатор пересчитывает каждый хеш и проверяет подписи локально.

```
timelayer-verifier verify cert.tlcert bundle.tlbundle
# -> VALID FINAL        (exit 0)  подлинна и полна
# -> NOT VALID / UNVERIFIABLE (exit 1)  подделка, расхождение или нечитаемый ввод
```

> **Статус: сеть в строю.** Каждая квитанция подписана **кворумом независимых операторов на
> Ed25519** — ни один узел (включая нас) не может выписать её в одиночку. Алгоритм проверки
> полностью открыт (принцип Керкгоффса): безопасность держится на **приватных ключах**
> операторов, а не на секретности кода. Внешний аудит безопасности — в дорожной карте; до тех
> пор мы не утверждаем «неподделываема», только «подписана кворумом и проверяема офлайн».

## Что именно проверяется

Квитанция — это пара `cert.tlcert` (сертификат) + `bundle.tlbundle` (подтверждающие данные).
Верификатор:

1. **пересчитывает корень** = `BLAKE3(domain ‖ canonical_fields)` из собственного содержимого
   квитанции (дайджест документа, позиция в причинной цепочке, эмитент, nonce, дайджест
   доказательства replay/shadow, дайджест когорты) — он никогда не доверяет переданному ему корню;
2. проверяет **подписи кворума на Ed25519** над этим корнем от подписавшей когорты;
3. подтверждает, что квитанция **FINAL** (полна, а не частичное/промежуточное состояние).

Если всё сходится — печатает `VALID FINAL`. Любое расхождение — перевёрнутый байт, отсутствующая
подпись, поддельный сертификат — печатает `NOT VALID` (или `UNVERIFIABLE` для нечитаемого ввода) и завершается с ненулевым кодом.

- Подписи: **Ed25519** (RFC 8032). Хеш: **BLAKE3**. Сериализация: явные поля с префиксом длины.
- **Офлайн и самодостаточен:** ни файла реестра, ни сетевого вызова, ни сервера ключей. Можно
  выдернуть сетевой кабель — и он всё равно работает.

## Сборка из исходников

```bash
cargo build --release
# бинарник в target/release/timelayer-verifier
```

Исходники здесь — это **полный** оффлайн-верификатор: нужные ему крейты receipt/cohort/finality
вендорены в `crates/`. Скрытых зависимостей нет — то, что проверяет вашу квитанцию, это ровно то,
что вы можете прочитать и скомпилировать.

## Использование

```bash
timelayer-verifier verify <cert.tlcert> <bundle.tlbundle>
```

Код выхода `0` = `VALID FINAL`, `1` = `NOT VALID` / `UNVERIFIABLE`.

## Тест-векторы (`testvectors/`)

```bash
timelayer-verifier verify testvectors/valid/cert.tlcert  testvectors/valid/bundle.tlbundle   # -> VALID FINAL
timelayer-verifier verify testvectors/forged/cert.tlcert testvectors/forged/bundle.tlbundle  # -> NOT VALID
```

`forged/` — это cert одной настоящей подписанной квитанции в паре с bundle другой: читаемая,
но расходящаяся «трансплантация», каноническая попытка подделки — и она отбивается как **NOT VALID**.
Оба вектора — настоящие квитанции `tlbundle/2`, отчеканенные живой сетью.

## Прозрачность ключей операторов (`pubkeys/`)

`pubkeys/epoch-2/` публикует публичные ключи текущих операторов на Ed25519, чтобы любой мог
независимо увидеть, какие ключи в сети. **Верификатору эти файлы не нужны** — квитанция
самодостаточна — они опубликованы исключительно для прозрачности и перекрёстной сверки.

## Модель угроз

- **Что доказывает квитанция.** Что данная пара `cert.tlcert` + `bundle.tlbundle` внутренне
  согласована (BLAKE3-корень), несёт маркер `FINAL` и подписана кворумом опубликованных ключей
  операторов (Ed25519) именно над этим содержимым — проверяемо офлайн, без сети, key server и
  обращения к roster.
- **Что она не доказывает.** *Истинность* содержимого. Квитанция доказывает, что кворум заверил
  этот конкретный документ, а не что утверждения в документе верны.
- **Компрометация ключа оператора.** Одного ключа недостаточно. `VALID FINAL` требует кворума
  подписей от *разных* независимых операторов, поэтому один скомпрометированный ключ не позволяет
  подделать квитанцию. Ключи ротируются публикацией новой epoch.
- **Кворум недоступен.** Fail-closed: если кворум не может подписать, квитанция не выдаётся — вы
  получаете ничего, а не ложную квитанцию. Уже выданные квитанции остаются проверяемыми офлайн
  всегда (самодостаточны).
- **Потеря квитанции.** Квитанция — это то, что вы храните; сеть не хранит ваше содержимое (от вас
  уходит только хеш). Если квитанция потеряна — перезаверьте действие, чтобы получить новую.
